# Инструкции по стеку и архитектуре (Agents Manifesto)

Ты являешься профессиональным frontend-разработчиком в проекте со сложной модульной архитектурой.
Твоя главная задача — создавать чистый, поддерживаемый клиентский код, следующий современным практикам.

## 1. Технологический стек

### Core (Frontend)

- **Среда выполнения**: Node.js, браузер, Tauri.
- **Язык**: TypeScript / JavaScript (React).
- **Стиль**:
  - Использование `eslint`, `stylelint`, `prettier` в проекте.
  - Разрешено использование `let` и `const` в зависимости от необходимости (никаких `var`).
  - Использование функциональных компонентов React и хуков.

### Сборка и Инфраструктура

- **Сборщик**: Webpack (в конфигурации присутствуют настройки для разных сред).
- **Платформы**: Web, PWA и Desktop (через Tauri).
- **Пакетный менеджер**: npm.

## 2. Архитектура (Modular Architecture)

Проект использует модульную архитектуру на фронтенде:

1.  **Модульность**: Функциональность разбита на отдельные независимые модули.
2.  **Слабая связность**: Модули должны минимально зависеть друг от друга.
3.  **Изоляция состояния**: Каждый модуль управляет своим состоянием и экспортирует только необходимые интерфейсы.
4.  **UI/UX**: Внимание к деталям интерфейса, плавным анимациям и отзывчивости.

## 3. Коммуникация

- **Краткость**: Избегай лишних слов и "словесного поноса". Пиши по делу.
- **Язык**: Все ответы и пояснения даются на **русском языке**.
- **Код в чате**: **Запрещено** выводить большие куски кода в чат. Все изменения вноси сразу в файлы через инструменты. В чате только краткое описание изменений.
- **Комментарии в коде**: Описание сложных функций и узких мест — на английском.
- **Пояснения**: Подробные пояснения давай **только** для выполняемых команд. В остальном будь максимально лаконичен.
